# Ответы на вопросы

## 1. Распространение событий в JavaScript (Event Propagation)

Когда в браузере происходит событие (например, клик), оно не обрабатывается сразу одним элементом. Событие проходит через несколько этапов — это и называется распространением событий.

### Фазы распространения событий

1. **Фаза погружения (capturing)**
   Событие идёт сверху вниз: от `document` к нужному элементу.

2. **Целевая фаза (target)**
   Событие попадает в элемент, по которому кликнули.

3. **Фаза всплытия (bubbling)**
   Событие поднимается обратно вверх: от элемента к `document`.

По умолчанию обработчики работают на фазе всплытия.

### Зачем это нужно на практике

- можно вешать один обработчик на родителя вместо множества обработчиков на дочерние элементы (делегирование событий)
- можно перехватывать события раньше или позже
- можно останавливать распространение события (`stopPropagation`)

Пример:

- список элементов (`ul > li`)
- один `click` на `ul`
- определяем, по какому `li` кликнули


## 2. Promise и асинхронность в JavaScript

Promise — это объект, который представляет результат операции, которая завершится в будущем. Например, HTTP-запрос.

### Состояния Promise

1. pending — ожидание результата
2. fulfilled — успешно выполнен
3. rejected — произошла ошибка

Promise можно обрабатывать с помощью:

- `.then()` — при успехе
- `.catch()` — при ошибке
- `.finally()` — в любом случае

### Другие способы работы с асинхронным кодом

- `async / await` — более читаемый синтаксис поверх Promise
- Observable (RxJS) — используется в Angular для потоков данных

### Event Loop

Event Loop — это механизм, который позволяет JavaScript работать с асинхронным кодом, несмотря на то, что он однопоточный.

Он:

- выполняет синхронный код
- обрабатывает очередь задач (Promise, события, таймеры)
- не блокирует интерфейс


## 3. ООП и его реализация в JavaScript

ООП (объектно-ориентированное программирование) — это подход, при котором программа строится из объектов.

### Основные принципы ООП

1. Инкапсуляция
   Скрытие внутренней логики и предоставление только нужного интерфейса.

2. Наследование
   Один класс может наследовать свойства и методы другого.

3. Полиморфизм
   Один и тот же метод может работать по-разному в разных классах.

4. Абстракция
   Выделение только важных характеристик объекта.

### ООП в JavaScript

В JavaScript ООП реализовано через:

* прототипы
* ключевое слово `class`

Пример:

* `class Movie`
* `class Service`
* `extends`

JavaScript не является классическим ООП-языком, но поддерживает все основные принципы.


## 4. Как браузер обрабатывает URL

Процесс начинается с момента ввода URL в адресную строку.

Основные шаги
1. Проверка кэша и DNS
2. Определение IP-адреса сервера
3. Установка соединения (HTTP / HTTPS)
4. Отправка HTTP-запроса
5. Получение HTML-документа
6. Загрузка CSS, JS, изображений
7. Построение DOM и CSSOM
8. Отрисовка страницы

Что ускоряет процесс
- кэширование
- CDN
- сжатие (gzip, brotli)
- HTTP/2, HTTP/3

Возможные проблемы
- безопасность XSS, CORS
- междоменное взаимодействие ограничения CORS
- медленные запросы: блокирующий JS

Браузер решает эти проблемы с помощью политик безопасности и оптимизаций загрузки.
